# Projektregeln

Dies ist eine Vanilla-HTML/CSS/JavaScript-App für einen Wochenplaner.

## Wichtig
- Keine Frameworks hinzufügen, außer ausdrücklich gewünscht.
- Bestehende Funktionen nicht verändern, wenn der Auftrag nur Refactoring ist.
- Kleine, sichere Änderungen bevorzugen.
- Nach Änderungen prüfen, ob bestehende IDs, Klassen und Event Listener weiterhin passen.
- index.html bleibt im Root.
- style.css enthält Styling.
- app.js enthält die Hauptlogik.
- api/ ist nur für Backend-/Serverless-Funktionen gedacht.

## Arbeitsweise
- Vor größeren Änderungen zuerst einen kurzen Plan schreiben.
- Nie mehrere große Features gleichzeitig umbauen.
- Bei Refactoring keine Feature-Änderungen einbauen.
- Bestehende localStorage-/Supabase-Datenstruktur nicht ohne Migration ändern.
